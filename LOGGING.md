# Логирование бота и связанных сервисов

## Файлы логов

| Файл | Назначение | Где создаётся |
|------|------------|---------------|
| **bot.log** | Диалоги, команды, ошибки бота (Telegram, DeepSeek, кнопки) | `bot.py` при `LOG_TO_FILE=True` |
| **payments.log** | Оплаты: создание заказа, ссылки, ошибки Robokassa (в процессе бота) | `bot.py` при `LOG_TO_FILE=True` (логгер `robokassa_integration`) |
| **robokassa.log** | Result URL, Success/Fail, уведомления после оплаты (сервер Robokassa на ВМ) | `robokassa_server.py` при запуске сервера |
| **digest.log** | Дайджест групповых оплат: запуск по расписанию, отправка в Telegram | `send_group_digest.py` при запуске скрипта |

Файлы создаются в текущей рабочей директории процесса (для бота и сервера — обычно каталог проекта или домашний каталог пользователя сервиса).

## Ротация и лимиты

Для каждого файла включена **ротация** (RotatingFileHandler):

- **Максимальный размер одного файла:** 5 MB (5·1024·1024 байт).
- **Число резервных копий:** 2 (хранятся `файл.1`, `файл.2`).

Настраиваемые константы:

- В `bot.py`: `LOG_BOT_MAX_BYTES`, `LOG_BOT_BACKUP_COUNT`, `LOG_PAYMENTS_MAX_BYTES`, `LOG_PAYMENTS_BACKUP_COUNT`.
- В `send_group_digest.py`: `LOG_DIGEST_MAX_BYTES`, `LOG_DIGEST_BACKUP_COUNT`.
- В `robokassa_server.py`: параметры в коде при создании `RotatingFileHandler` для `robokassa.log`.

## Что происходит при исчерпании лимита размера файла

1. Текущий лог-файл достигает **maxBytes** (5 MB).
2. Handler **ротирует** файл:
   - Текущий `файл.log` закрывается и переименовывается в `файл.log.1`.
   - Если уже существовал `файл.log.1`, он переименовывается в `файл.log.2`.
   - Если уже существовал `файл.log.2`, он **удаляется** (храним только 2 копии).
   - Создаётся новый пустой `файл.log`, запись продолжается в него.
3. Никаких ошибок приложения при этом не происходит: логирование продолжается в новый файл. Старые данные остаются в `файл.log.1` и `файл.log.2` до следующей ротации.

Итог: при исчерпании лимита старый файл сохраняется с суффиксом `.1`, ещё более старый — `.2`, самый старый удаляется; запись идёт в новый `файл.log`. Потери логов при нормальной работе нет, пока не превышен суммарный объём (текущий + 2 копии).

## Включение записи в файл для бота

В `.env` или в коде задайте:

- `LOG_TO_FILE=True` — бот будет писать в `bot.log` и (для вызовов из бота) в `payments.log`.

При `LOG_TO_FILE=False` (по умолчанию) вывод идёт только в консоль (stdout/stderr).
